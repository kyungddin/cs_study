# 도커 공부하기


## System Configuration
- 하드웨어 - 커널 - 운영체제 - 셸 - 애플리케이션
    - 셸은 OS와 사용자 사이에서 다리 역할을 함. 즉 운영체제와 상호작용을 돕는 친구
- 네임스페이스: 프로세스를 실행할 때 시스템 리소스를 분리해서 실행할 수 있도록 도와주는 기능
    - 리눅스 네임스페이스 ex: pid, net, mnt 등등..


## 도커 기초 지식
- 도커: 컨테이너라고 부르는 운영체제 수준의 가상화 방식으로 소프트웨어를 배포하는 방식을 사용하는 Paas제품
    - Paas: Platform as a Service, 즉 App/데이터 단계만 사용자가 관리하는 서비스
- Hierarchy: 하드웨어 - 운영체제 - 도커 - 컨테이너들(with 애플리케이션, 바이너리/라이브러리)


## 가상화 개념
- 호스트 가상화: 하드웨어 - 호스트 OS - 하이퍼바이저 - VM들(with 게스트 OS, 바이너리/라이브러리, App)
- 하이퍼바이저 가상화: 하드웨어 - 하이퍼바이저 - VM들
- 컨테이너 가상화: 하드웨어 - OS - 도커 - 컨테이너들


## Namespace vs Cgroup
```
| 구분        | Namespace     | cgroup       |
| --------- | ------------- | ------------ |
| 목적        | 격리            | 제한/관리        |
| 대상        | 보이는 환경        | 자원 사용량       |
| 느낌        | “서로 다른 컴퓨터처럼” | “CPU/RAM 배급” |
| Docker 역할 | 컨테이너 분리       | 컨테이너 성능 제어   |
```
- Namespace는 한 OS 내에서 프로세스를 독립시키는 것
- cgroup은 한 프로세스 내에서 자원을 독립시키는 것, 리눅스 커널?
- 둘 다 docker 컨테이너의 독립을 위해 필요한 요소임
