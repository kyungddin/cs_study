# HTTP
HTTP 프로토콜에 대해 정리


## Conecpt
- 웹의 애플리케이션 계층 프로토콜
    - 그러나, 현재는 범용 통신 프로토콜임
    - 웹
        - 단일 URL로 지정할 수 있는 하나의 파일인 객체로 구성
        - 대부분의 웹 페이지는 기본 HTML 파일과 여러 객체로 구성
        - 기본 HTML 파일은 페이지 내부의 다른 객체를 URL로 참조

    - 웹서비스
        - HTTP를 통해 다른 프로그램이 사용할 수 있게 기능이나 데이터를 제공하는 서버

    - World Wide Web (WWW)
        - HTTP + URL + HTML + 브라우저로 이루어진 하이퍼텍스트 문서 시스템
        - 하이퍼텍스트 문서를 URL로 연결하고 HTTP로 주고받는 시스템

- HTTP는 TCP를 전송 프로토콜로 사용

- 서버는 클라이언트에 관해 어떤 정보도 저장 X
    - 이런 점에서, HTTP 서버는 비상태 프로토콜을 사용한다고 한다

- 비지속연결 vs 지속연결
    - 비지속연결: 서버가 객체를 보낸 후 각 TCP 연결이 끊어진다
    - 지속연결: TCP 연결이 객체 전송 후에도 유지된다
        - 일정 기간동안 사용되지 않을 때 연결이 끊어진다

- HTTP와 같은 텍스트 기반 프로토콜은 CRLF로 줄바꿈을 구분한다
    - CR(Carriage Return) = \r
    - LF(Line Feed) = \n


## HTTP 메시지 포맷
- HTTP 요청 메시지의 일반 포맷
    - 요청라인 = 방식 + SP + URL(리소스) + SP + 버전 + CRLF
    - 헤더라인 = 헤더 필드 이름 + SP + 값 + CRLF
    - 공백라인 = CRLF
    - 개체몸체: POST 방식에서 사용 (ex: 사용자가 검색 단어 넣기)

    - C++ 프로그래밍 시
        - 요청라인은 반드시 구현해야 한다
        - 헤더는 Host, Content-Length, Conetnet-Type 정도면 충분
        - CRLF는 말 안해도 필수
