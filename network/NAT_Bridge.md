# NAT과 Bridge의 차이
둘 다, 외부와 내부 네트워크 호스트를 연결하는 기술


## NAT(Network Address Translation)
- 내부IP의 경우 변환을 거쳐 외부IP로부터 숨긴다
- NAT 변환 테이블을 거쳐 내부IP는 외부IP와 완전히 독립된 IP주소를 가진다
- 따라서 NAT은 보통 보안이 중요한 상황에서 쓰인다
- IP를 변환하는 기술 때문에 Layer 3


## Bridge
- 내부IP와 외부IP의 네트워크를 연결해버린다
- 따라서 외부와 내부는 같은 Subnet을 공유한다
- 따라서 한쪽을 해킹해서 네트워크 내부로 침입하는게 가능하므로 보안에는 취약
    - 회사의 경우 브리지를 통해 회사 네트워크를 털 수 있기 때문에 브리지는 일반적으로 차단한다
- 다만 직관적이고 단순하다
- Layer2


## 포트포워딩
- 일반적으로 LAN에서는 각 기기별로 IP를 공유한다
- 그러나 외부와 연결될 때는 게이트웨이를 거치고, 그 게이트웨이를 거치는 과정에서 NAT을 사용하는 것이 일반적이다
- 따라서 외부에서는 어떤 네트워크에 접속할 때 내부 기기들의 IP까지는 알 수 없다
- 그래서 Port를 사용하며, 게이트웨이에서는 포트 포워딩을 통해 특정 기기로의 연결을 가능하게 해준다
