## 네트워크의 기본 구조

- 네트워크의 기본 구조는 그래프, 즉 Node와 Edge로 이루어짐
- 호스트: 가장자리 노드이자 종단 시스템
- 서버: 서비스를 제공하는 호스트
- 클라이언트: 서버에게 서비스를 요청하고 응답을 제공받는 호스트
- 네트워크 장비: 중간 노드로 호스트 간 주고받는 정보를 안전하게 전송되게 돕는 역할
    - 이더넷 허브, 스위치, 라우터, 공유기 등이 있다


## 범위에 따른 네트워크 분류

- LAN(Local Area Network): 가까운 지역을 연결하는 근거리 통신망 (가정, 기업, 학교)
- WAN(Wide Area Network): 먼 지역을 연결하는 광역 통신망
    - 서로 다른 LAN에 속한 호스트와 메시지를 주고 받을 때는 WAN이 필요하다
    - WAN은 ISP(Internet Service Provider)라는 인터넷 서비스 업체가 구축하고 관리한다
        - ex: KT, LG유플러스


## 메시지 교환 방식에 따른 네트워크 분류

- 회선 교환 방식: 메시지 전송로인 회선을 설정하고 메시지를 주고받는 방식
    - 특정 호스트들이 회선을 점유하고 메시지를 보내지 않는다면 효율이 낮다는 치명적 단점이 있다
- 패킷 교환 방식: 
    - 패킷: 패킷 교환 네트워크상에서 송수신되는 메시지의 단위
    - 패킷 스위치가 최적의 경로를 설정하며, 이에 따라 중간 경유 노드가 달라질 수 있다
        - 패킷 스위치의 예로는 라우터가 있다
    - 패킷은 페이로드라는 메인 데이터에 헤더/트레일러가 포함될 수 있다


## 주소와 송수신지 유형에 따른 전송 방식

- 유니캐스트: 하나의 수신지에 메시지를 전송하는 방식으로, 일대일로 메시지를 주고 받음
- 브로드캐스트: 자신을 제외한 네트워크상의 모든 호스트에게 전송하는 방식


## 프로토콜

- def: 노드 간에 정보를 올바르게 주고받기 위해 합의된 규칙이나 방법
- 프로토콜은 저마다의 목적과 특징이 있다. 즉, 프로토콜마다 패킷의 헤더 내용이 달라질 수 있다
    - 헤더가 없는 프로토콜도 있다


## 네트워크 참조 모델 (Network Reference Model)

- 장점
    1. 네트워크 구성과 설계가 용이하다
